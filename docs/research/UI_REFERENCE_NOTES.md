# UI Reference Notes

## 確認済み

- 公式サイトでは、シリーズの基本は「コトバを教えながら、おしゃべりを楽しむ」お話しゲームとして説明されている。
- 外部プレイ記録では、文字入力で単語を教えたあと、「何を表す言葉か」「好きか」などの選択質問に答え、その単語が後日の会話へ混ざる流れが説明されている。
- PocketStation連携により、持ち歩き、外でも会話や言葉学習をする構造がある。

## 推測

- PWAではPocketStationそのものは再現せず、IndexedDB保存とoffline起動を「持ち歩ける記憶」の抽象代替にする。
- UIは、中央のキャラクター表示、下部の会話表示、選択肢による応答という構造を重視する。

## 不明

- 原作内部の正確なテンプレート選択条件、メモリカード保存形式、単語上限、質問テーブルの完全仕様は未確定。

## 実装に使う抽象

- 単語入力
- カテゴリ質問
- 好き嫌い/感情質問
- 関連質問
- confidenceと使用回数による再利用頻度調整
- blocked/sensitive語を通常会話に出さない制御

## 実装に使わないもの

- 原作キャラクター名
- 原作固有台詞
- 原作画像、音声、ロゴ
- 原作のUI文言そのもの
