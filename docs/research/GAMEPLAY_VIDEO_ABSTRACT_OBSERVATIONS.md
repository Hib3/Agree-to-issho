# プレイ動画から得た抽象観察

参照日: 2026-07-11

対象: ユーザー提供のローカル動画 `testPlay.mp4`

方法: `ffmpeg` で2秒間隔の全体フレームと、後半130秒以降の1秒間隔フレームを `research_private/` に生成して目視確認した。

動画、抽出フレーム、原文台詞、登場語彙はリポジトリへ保存しない。本書にはファンメイド実装へ転用可能な抽象構造だけを記録する。

## 確認済み

- 単語入力の後に、種類、好み、場面、人物関係などを段階的に質問する画面がある。
- ユーザーが選んだ分類が一般常識とずれていても、その回答を後の会話で使用する例がある。
- キャラクターは、教えられた複数の単語を一つの小話の異なる役割へ入れて再利用する。
- 文の形は読める状態を保ちつつ、場所、入れ物、原因、結果などの接続が少しずれる例がある。
- 一つの自発的な話題が複数の吹き出しページに続き、途中または末尾で選択肢が出る。
- 吹き出しの本文は一度に全表示されず、時間経過に合わせて段階的に増える。
- 選択後にはキャラクターの反応ページがあり、その後に話題を閉じる。

## 推測

- 内部では、単語本文だけでなくカテゴリ、好み、関係、利用場面に相当する属性を保持している可能性が高い。
- 発言生成は自由作文ではなく、場面フレームの役割へ型付き単語を割り当てる方式と考えると観察結果を説明しやすい。
- 面白いずれは全文のランダム破壊ではなく、一つの役割または因果関係だけを弱く誤接続することで作られている可能性が高い。
- 選択回答は、単語そのものの意味修正に加えて、単語間の関連の強化または分離に使われている可能性がある。

## 不明

- 原作内部のデータ構造、重み、乱数、テンプレート数は不明。
- 文字送りが完全な自動送りか、プレイヤー入力による送りかは、操作表示が動画にないため不明。
- 一般語彙の基礎辞書とユーザー学習語彙を内部でどう分離しているかは不明。
- 誤接続を意図的に生成しているのか、ユーザー分類を忠実に使った結果なのか、その比率は不明。

## PWAへの抽象実装

- `WordFrame` を単語記憶の正本とし、カテゴリ、感情、場面、関係、理解度を保持する。
- 複数語会話では、人物、場所、行動、物、食べ物、感情、時間の役割を持つ場面フレームを使う。
- 通常会話はカテゴリ適合を優先し、ずれ会話は一つの役割だけを取り違える。
- ずれた文でも、引用された単語の周囲の助詞と文構造は壊さない。
- 自発会話は `導入 -> 小話 -> 確認 -> 反応 -> 終了` の保存可能なページ列として扱う。
- 文字送りは段階表示し、最初の操作で現在ページを全文表示、次の操作で次ページへ進む。
- ユーザーが関連を肯定した場合は単語間リンクを強め、否定した場合はその組み合わせを分離する。

## 持ち込まないもの

- 動画に出たキャラクター、部屋、UI、語彙、台詞、選択肢文言、固有設定。
- 原作の辞書、質問文、場面名、画像、音声、動画フレーム。
- 未確認の内部仕様を原作アルゴリズムと断定した実装。
